CASE FINDER – PUBLIC USER VIEW (SAFE & PROFESSIONAL)

Prompt:

Design and implement a Public Case Finder page for a Lawyer Case Management Web Application.

This page is intended only for public users, not lawyers or court officials. The goal is to help users understand case categories, explore publicly available case references, and find appropriate legal guidance, without exposing confidential or official court data.

Purpose:

Guide public users to identify the type of legal case they may have

Show publicly available case references and judgment summaries

Encourage users to book an appointment or request consultation

Avoid giving legal advice or showing sensitive data

Page Header:

Title: Case Finder
Subtitle: Find relevant case information and legal guidance based on publicly available records.

Display a visible disclaimer below the header:

This tool is for informational purposes only and does not constitute legal advice. Data shown is based on publicly available references.

Search & Filter Section:

Allow users to search using safe and non-sensitive filters only:

Case Category (Civil, Criminal, Family, Corporate, Property, Labour, Consumer, Others)

Court Level (Supreme Court, High Court, District Court)

Year (dropdown)

Keyword Search (placeholder: Enter case topic or reference keyword)

❌ Do NOT allow searching by:

Personal phone numbers

Full party personal details

Confidential advocate data

Public Case Results Section:

Display results as summary cards, not full records.

Each card should show:

Case Reference ID (example: SC-2024-001)

Case Category

Court Name

Year

Current Status (example: Judgment Reserved / Hearing Scheduled)

Each card should include:

“View Summary” button

“Consult a Lawyer” CTA

Case Summary View (Optional Modal / Page):

Show only high-level, public-safe information:

Case category

Court

Year

Public judgment summary (if available)

Current procedural status

❌ Do NOT show:

Party personal contact details

Advocate private details

Internal court notes
